Prueba de Concepto: Mi Enfoque a CloudOps

De Scripts a Infraestructura como Código

En mi rol actual, automatizo tareas con scripts (Python, Shell). Entiendo que la Infraestructura como Código (IaC) es la evolución de esta disciplina en la nube. Aunque no la uso a diario, mi formación en DevOps me dio bases sólidas.

Para demostrar cómo aplico estos conceptos, usé Terraform para definir un componente. Esto refleja mi capacidad para llevar mis habilidades de automatización al siguiente nivel en un entorno cloud.

Mi experiencia con Jenkins se centra en automatizar tareas puntuales como ETLs. Para esta PoC, elegí GitHub Actions para demostrar mi capacidad de adoptar rápidamente herramientas modernas e integradas de CI/CD.

Orquestación con Kubernetes

Mi experiencia práctica con Kubernetes viene de trabajar con microk8s, lo que me ha dado una base sólida en la gestión de manifiestos y el ciclo de vida de las aplicaciones. Por eso, me siento cómodo demostrando mi conocimiento.

Este manifiesto se valida en el pipeline usando kubectl --dry-run, una práctica profesional que asegura la viabilidad del despliegue antes de tocar un clúster productivo como EKS.

deployment.yaml

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.21.0
        ports:
        - containerPort: 80
